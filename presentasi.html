Tantangan Klasik Enterprise IT

Fenomena "The Silo Effect" di tim besar:

  • Fullstack Dev: Fokus UI/API *performance*.
  • UI/UX Designer: Fokus estetika & *user experience*.
  • Data Scientist: Fokus akurasi model ML.
  • Data Analyst: Fokus *insight* & pelaporan bisnis.
  • Data Engineer: Fokus optimasi *pipeline*.
  • Computer Vision Eng: Fokus akurasi deteksi *image*.
Spesialisasi yang terlalu dalam melahirkan Friction (Gesekan) integrasi, keterlambatan rilis, dan miskomunikasi bisnis.

Analogi Pabrik Mobil

Solusi: T-Shaped Developer

Konsep talenta yang menjadi kolaborator efisien.

  • Garis Vertikal (Depth):
    Spesialisasi teknis utama Anda (Deep Expertise).
  • Garis Horizontal (Breadth):
    Komunikasi lintas disiplin, pemahaman bisnis, dan empati pada tim lain.
T-Shaped Developer tidak asyik sendiri; mereka memastikan karya mereka relevan dengan tujuan besar perusahaan.

T-Shaped + AI = Superpower

  • AI sangat unggul di ranah Vertikal (Kalkulasi, *Syntax*, Pencarian solusi teknis dengan instan).
  • Namun AI sama sekali buta di ranah Horizontal (Empati *user*, konteks bisnis, dinamika tim, dan politik kantor).
  • Strategi: Delegasikan pekerjaan teknis berulang ke AI, bebaskan waktu Anda untuk memperluas kolaborasi.

Anatomi AI: Garbage In, Garbage Out

Mengapa AI terkadang berhalusinasi atau memberikan *code* yang salah?

  • AI bukan Google Search. AI butuh kepadatan Context Window.
  • Jika *prompt* Anda terlalu singkat, memori konteks AI kosong, sehingga ia menebak probabilitas secara acak.
  • Prinsip: Kualitas dan akurasi *output* AI berbanding lurus dengan kelengkapan *parameter* yang Anda berikan.

Minim Konteks = Halusinasi

Teori Prompting: Membatasi AI

Teknik mengendalikan attention mechanism dari Large Language Models.

1. Zero-Shot Prompting

Memberi instruksi tanpa contoh. Tingkat halusinasi tinggi untuk *task* logis.

"Buatkan fungsi Regex untuk memvalidasi nomor telepon."
(Hasil: Format USA/acak).

2. Few-Shot Prompting

Instruksi + Contoh Pola (Input-Output). Akurasi meningkat drastis.

"Validasi nomor telepon Indonesia.
Contoh 1: Input '0812' -> Valid.
Contoh 2: Input '+123' -> Invalid.
Sekarang buatkan fungsi Regex-nya."

Framework C.R.E.A.T.E

Panduan bagi *Developer* agar AI tidak berhalusinasi.

C Context

"Saya *Data Engineer* di proyek migrasi DB."

R Role

"Bertindaklah sebagai Senior DB Administrator."

E Expectation

"Saya ingin merancang skema relasional tabel."

A Action

"Buatkan query DDL beserta *index optimization*."

T Tech Stack / Tone

"Gunakan PostgreSQL versi terbaru."

E Example

"Ini contoh JSON lama kita sebagai referensi: {...}"

Bedah Kasus: Prompting SQL

❌ Prompt "Ngawang"

"Bikinin query SQL buat narik data user aktif di bulan ini dong."

Fatal: AI terpaksa menebak nama tabel dan definisi "aktif". Pasti terjadi *error* saat kode disalin ke lokal.

✅ Prompt Profesional

"Tabel: trx_users (id: uuid, created_at: timestamp, status: varchar).

Buatkan query PostgreSQL untuk total user aktif (status = 'A') di bulan Juni 2026. Prioritaskan optimasi query."

Hasil: Kode siap *production* karena batasan sistem dikunci dengan ketat.
